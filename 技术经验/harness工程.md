- OpenSpec 负责：先把“要做什么”说清楚
- Superpowers 负责：让 agent 默认按⼯程纪律⼯作
- GSD 负责：把复杂任务拆进⼲净上下⽂中执⾏
- OMC 负责：把 Claude Code 组织成团队式执⾏系统
- ECC 负责：给 Harness 补技能、记忆、安全、验证和学习能⼒
- Trellis 负责：把 specs、tasks、workspace 变成统⼀⼯作流⻣架

如果再把它们按定位压缩⼀句，可以更清楚地看出差异：

OpenSpec、Superpowers、GSD 则更偏向单层补位：分别聚焦规范、⼯程技能、上下⽂与阶段执⾏  
OMC、ECC、Trellis 更偏体系型⽅案，只是各⾃覆盖的主轴不同

- ECC 更像在补 skills、memory、安全、验证、学习等能⼒，覆盖⾯更⼴，也更偏体系型增强系统
- Trellis 在 specs / tasks / workspace / project memory 上更完整，更像⻓期⼯作流⻣架
- OMC 在 Claude Code 的 team-first orchestration 上更像⼀套成体系的编排套件分层搭建、按缺⼝补位的问题 ，⽽不是单⼯具选择题。
